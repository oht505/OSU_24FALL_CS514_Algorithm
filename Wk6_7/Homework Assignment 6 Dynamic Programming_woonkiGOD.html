1. Write a Python function called `editDistance' that takes two strings as inputs and returns the edit distance between the two strings.

(a) Python Implementation

def editDistance(x, y):
    if len(x)==0 or len(y)==0 or x == y :
        return 0

    m = len(x)+1
    n = len(y)+1

    D = [[0 for i in range (n)] for j in range (m)]

    for i in range (m):
        D[i][0] = i
    for j in range (n):
        D[0][j] = j

    for i in range (1,m):
        for j in range (1,n):
            if x[i-1] == y[j-1]:
                D[i][j] = D[i-1][j-1]
            else:
                insertion = 1+D[i][j-1]
                deletion = 1+ D[i-1][j]
                replacement = 1+ D[i-1][j-1]

                D[i][j] = min (insertion, deletion, replacement)

    return D[len(x)] [len(y)]

def LCS(x, y):
    if len(x) == 0 or len(y) == 0:
        return None
    elif x == y:
        return [x]

    m = len(x)
    n = len(y)

    D = [[0 for i in range(n+1)] for j in range(m + 1)]
    

    lcs_length = 0
    lcs = []

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if x[i - 1] == y[j - 1]:
                D[i][j] = D[i - 1][j - 1] + 1

                if D[i][j] > lcs_length:
                    lcs_length = D[i][j]
                    lcs = [x[i - lcs_length:i]]

                elif D[i][j] == lcs_length:
                    substring = x[i - lcs_length:i]

                    if substring not in lcs:
                        lcs.append(substring)

    return lcs

#d is a possible locations in a straight line to the south of their Headquarters.
#p is a potential profits
#k is minimal distance between two shops
def bigBucks(d, p, k):
    n = len(d)
    dp = [0 for i in range (n)]
    sorted_pair = sorted(zip(d, p)) #sort just in case.
    d, p = zip(*sorted_pair) 

    dp[0] = p[0]

    for i in range (1,n):
        dp[i] = dp[i-1]

        while i-1>=0:
            if d[i] - d[i-1] >= k:
                dp[i] = max(dp[i], dp[i-1] + p[i])
                break
            i = i-1

        dp[i] = max(dp[i] , p[i])

    return dp[n-1]

4. Rope cutting problem

Bellman-equation:

  • The cost of cutting the sub-rope from iii﻿ to jjj﻿ includes the costs of two smaller sub-ropes, one from iii﻿ to kkk﻿ and the other from kkk﻿ to jjj﻿, where kkk﻿ is an intermediate cut point between iii﻿ and jjj﻿. Assume that this is bottom-up approach where we start calculating from the smallest sub-rope possible. By this assumption we would already have minimum cost of sub-rope iii﻿ to kkk﻿ and kkk﻿ to jjj﻿. And we also need to add cost by cutting j−ij-ij−i﻿ length rope.
  • Thus the bellman equation can be formulated as:

    Cost[i][j]=min⁡i<k<j(Cost[i][k]+Cost[k][j]+(cuts[j]−cuts[i]))Cost[i][j] = \min_{i < k < j} \left( Cost[i][k] + Cost[k][j] + (cuts[j] - cuts[i]) \right)Cost[i][j]=mini<k<j​(Cost[i][k]+Cost[k][j]+(cuts[j]−cuts[i]))﻿

Bottom-up Approach

  • Table Structure

    To solve this problem, we use a 2-D dynamic programming table D:

    • D[i][j]D[i][j]D[i][j]﻿ will store the minimum cost of cutting the rope between cuts[i]cuts[i]cuts[i]﻿ and cuts[j]cuts[j]cuts[j]﻿.
  • Table Size

    To store every sub-rope’s cost, the size of the table D is determined by the number of cuts plus two.

    ((len(X)+2)∗(len(X)+2))((len(X)+2) * (len(X)+2))((len(X)+2)∗(len(X)+2))﻿

  • Table Initialization
    • Add the start (0) and end (n) of the rope to the cuts list, so the cuts are inclusive of the rope's endpoints.
    • Sort the cuts list to ensure that we process cuts in the correct order.
    • Set D[i][i] = 0 for all i because a segment with no cuts has zero cost.
    • Initialize D[i][j] to infinity for pairs [i, j] where there’s no direct segment (i.e., when we are calculating minimum costs over multiple cuts).
  • Iterative Structure to Fill the Table

    Now we calculate bottom-up way starting from minimum length of sub-rope 2( 1 cannot be cut).

    • For each length sub-rope, the starting point iii﻿ is from 0 up to (number of cuts - sub_rope size), ensuring that there’s room for a sub-rope.
    • For each starting point iii﻿, the endpoint jjj﻿ is defined as i+i + i+﻿sub_rope length.
    • And as shown in bellman equation, we loop through intermediate value kkk﻿ and calculate each points minimum cost:

      Cost[i][j]=min⁡i<k<j(Cost[i][k]+Cost[k][j]+(cuts[j]−cuts[i]))Cost[i][j] = \min_{i < k < j} \left( Cost[i][k] + Cost[k][j] + (cuts[j] - cuts[i]) \right)Cost[i][j]=mini<k<j​(Cost[i][k]+Cost[k][j]+(cuts[j]−cuts[i]))﻿

Time Complexity

  • Since this algorithm has three nested loops: one for the sub-rope’s length, one for the starting point iii﻿, and one for choosing the cut point kkk﻿. Thus the time complexity is O(n3)O(n^3)O(n3)﻿, where n=n = n=﻿ number of cuts required.

def RopeCutting(n, X):
    X = [0] + sorted(X) + [n]
    cut_len = len(X)

    D = [[0 for i in range (cut_len)] for j in range (cut_len)]

    for subRope in range(2, cut_len):
        for i in range(cut_len - subRope):
            j = i + subRope
            D[i][j] = float('inf')

            for k in range(i + 1, j):
                cost = D[i][k] + D[k][j] + X[j] - X[i]
                D[i][j] = min(D[i][j], cost)

    return D[0][cut_len - 1]
